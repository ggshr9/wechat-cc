# Handoff — Pending work after overnight VoxCPM2 deploy (2026-04-22)

Context: v1.1.0-rc.1 was tagged and VoxCPM2 was deployed on the Mac overnight. This doc captures what remains so the next session (possibly on a different machine) can pick up without losing state.

## Immediate user actions (on the Mac, ~30 seconds)

Until this is done, every Windows/other machine that wants to use voice needs the SSH tunnel up (`scripts/voxcpm-tunnel.ps1`). See `docs/ops/voxcpm2-deployment.md` for full rationale.

1. On the Mac: **System Settings → Privacy & Security → Local Network** → toggle Python ON (or accept the prompt next launch).
2. Edit `~/Library/LaunchAgents/com.wechat-cc.voxcpm.plist`: change `VOXCPM_HOST` from `127.0.0.1` to `0.0.0.0`.
3. `launchctl unload ~/Library/LaunchAgents/com.wechat-cc.voxcpm.plist && launchctl load ~/Library/LaunchAgents/com.wechat-cc.voxcpm.plist`.
4. Update `~/.claude/channels/wechat/voice-config.json` `base_url` to `http://100.64.249.44:8765/v1/audio/speech`.
5. SSH tunnel no longer needed from any machine.

After step 5, a fresh dev machine only needs: `git pull` + write its own `voice-config.json` (or call the `save_voice_config` tool from WeChat). No SSH-key bootstrap, no tunnel.

## Cross-machine migration notes

State that does NOT live in git and must be reestablished on a new machine:

- **SSH access to Mac** — the `~/.ssh/id_ed25519` with comment `claude-overnight-deploy@desktop-tsfk2l9` is on *this* Windows only. New machine: either copy that key over, or bootstrap a new one with the user's Mac password and append the new pubkey to `homebot:~/.ssh/authorized_keys`. Only needed if the new machine needs to admin the Mac — voice calls just need the server reachable.
- **`voice-config.json`** — lives at `~/.claude/channels/wechat/voice-config.json` (mode 0600), outside the repo. Current contents point at `http://127.0.0.1:8765/v1/audio/speech`. After the 0.0.0.0 fix above, new machines should write `http://100.64.249.44:8765/v1/audio/speech` instead (or use `save_voice_config` from WeChat).
- **SSH tunnel helper** — if the 0.0.0.0 fix isn't done yet, run `scripts/voxcpm-tunnel.ps1` before using voice.

## VoxCPM2 tuning ideas (not urgent)

- **Latency**: "晚安" took ~11s on first synth (cold cache, 2 chars). VoxCPM has `inference_timesteps=10` default; dropping to 5 halves diffusion cost. Subsequent synths were already faster (~3-5s warm). Worth testing if latency becomes a pain point.
- **Voice-bubble rendering**: See Spike 4 above. Until then, `reply_voice` deliveries arrive as tap-to-play file attachments, which still works but is uglier.
- **Cold-start time**: Model load is ~10-15s. launchd's `KeepAlive: true` means it stays warm once booted. The only cold path is Mac reboot.
