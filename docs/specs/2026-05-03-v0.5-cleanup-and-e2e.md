## 3. A. Ref + wireRef 契约

### 3.1 类型

```ts
// src/lib/lifecycle.ts (新增；与 Lifecycle / LifecycleSet 同文件)

/**
 * 延迟绑定容器 — 解决"循环构造依赖"。
 *
 * 典型场景：admin command handler 需要持有 polling lifecycle 的引用
 * 来调 stopAccount/running，但 polling lifecycle 自己又需要先把 pipeline
 * （内含 admin handler 闭包）当 onInbound 才能 register。
 *
 * 用法：
 *   const ref = new Ref<PollingLifecycle>('polling')
 *   const handler = makeAdmin({ pollHandle: { running: () => ref.current?.running() ?? [] } })
 *   const lc = registerPolling({ ..., runPipeline })
 *   wireRef(ref, lc)
 */
export class Ref<T> {
  private _current: T | null = null
  constructor(public readonly name: string) {}

  get current(): T | null { return this._current }

  set(value: T): void {
    if (this._current !== null) {
      throw new Error(`Ref<${this.name}>: already set; cannot reassign`)
    }
    this._current = value
  }

  /**
   * 返回 T 而非 T | null。第一次访问时 ref 必须已 set，否则抛。
   * 给"承诺已 wire 的下游闭包"用，消除 ?. 链。
   */
  deref(reason: string = ''): T {
    if (this._current === null) {
      throw new Error(`Ref<${this.name}>: accessed before set${reason ? ` (${reason})` : ''}`)
    }
    return this._current
  }
}

/**
 * 一锤子 set + assert. main.ts 调用：
 *   wireRef(wired.refs.polling, pollingLc)
 *
 * 比 `wired.refs.polling.current = pollingLc` 更显式，
 * 且 Ref.set 内置的双重赋值检查会自动 fire。
 */
export function wireRef<T>(ref: Ref<T>, value: T): void {
  ref.set(value)
}
```

### 3.2 测试（追加到 lifecycle.test.ts）

4 个新用例：
- `Ref.current` 默认 null
- `Ref.set` 第一次成功，第二次抛 "already set"
- `Ref.deref` 未 set 时抛 "accessed before set"
- `wireRef(ref, v)` 等价于 `ref.set(v)`

### 3.3 接口改动 contract

```ts
// 旧 (main-wiring.ts WiredDeps):
refs: {
  polling: { current: PollingLifecycle | null }
  guard: { current: GuardLifecycle | null }
}

// 新 (wiring/index.ts WiredDeps):
refs: {
  polling: Ref<PollingLifecycle>
  guard: Ref<GuardLifecycle>
}
```

main.ts 改动：
```ts
// 旧:
wired.refs.guard.current = guardLc
// 新:
wireRef(wired.refs.guard, guardLc)
```

闭包内的 read 路径**不变**（仍 `ref.current?.x ?? fallback`）。`deref()` 接口本次只**预留**，不强制使用。

## 4. B. wiring/ 5 文件拆分

### 4.1 目录结构

```
src/daemon/wiring/
├── index.ts            (~40 LOC) — re-export wireMain 和 types，纯门面
├── side-effects.ts     (~60 LOC) — 4 个共用闭包工厂：fireMilestonesFor, recordInbound, maybeWriteWelcomeObservation, isolatedSdkEval
├── tick-bodies.ts      (~50 LOC) — buildTickBodies(deps): { pushTick, introspectTick }
├── pipeline-deps.ts    (~80 LOC) — buildPipelineDeps(opts, refs): { pipelineDeps: InboundPipelineDeps }
└── lifecycle-deps.ts   (~70 LOC) — buildLifecycleDeps(opts, ticks): { 6 个 lifecycle deps }
```

总 ~300 LOC（vs 当前 269）—— 略涨因为多了 5 个文件 header；接受。

### 4.2 各文件契约

**index.ts** — 纯组装：
```ts
export interface WireMainOpts { stateDir, db, ilink, accounts, boot, dangerously, log }
export interface WiredDeps { /* 8 dep structs + refs */ }

export function wireMain(opts: WireMainOpts): WiredDeps {
  const refs = {
    polling: new Ref<PollingLifecycle>('polling'),
    guard: new Ref<GuardLifecycle>('guard'),
  }
  const ticks = buildTickBodies(opts)
  return {
    ...buildPipelineDeps(opts, refs),
    ...buildLifecycleDeps(opts, ticks),
    refs,
  }
}
```
**不持有任何业务逻辑**。

**side-effects.ts** — store-construction 助手 + sdk eval：
```ts
export function makeFireMilestonesFor(deps: { stateDir, db }): (chatId: string) => Promise<void>
export function makeRecordInbound(deps: { stateDir, db }): (chatId: string, when: Date) => Promise<void>
export function makeMaybeWriteWelcomeObservation(deps: { stateDir, db }): (chatId: string) => Promise<void>
export function makeIsolatedSdkEval(): (prompt: string) => Promise<string>
```

每个工厂封装 store-construction + memoryRoot 路径计算。当前 main-wiring 里最重复的部分。

**tick-bodies.ts** — 真有 if 分支的 companion 逻辑：
```ts
export interface TickDeps { stateDir, db, ilink, boot, log }
export interface TickBodies {
  pushTick: () => Promise<void>
  introspectTick: () => Promise<void>
}
export function buildTickBodies(deps: TickDeps): TickBodies
```

**pipeline-deps.ts** — admin/mode/onboarding handler 装配 + 13 mw deps：
```ts
export function buildPipelineDeps(
  opts: WireMainOpts,
  refs: WiredDeps['refs'],
): { pipelineDeps: InboundPipelineDeps }
```

**lifecycle-deps.ts** — 6 个 lifecycle 的 deps（companion×2 / guard / sessions / ilink / polling / startup）：
```ts
export function buildLifecycleDeps(
  opts: WireMainOpts,
  ticks: TickBodies,
): {
  companionPushDeps, companionIntrospectDeps, guardDeps,
  sessionsDeps, ilinkDeps, pollingDeps, startupDeps,
}
```

### 4.3 main.ts 改动（最小）

```ts
// 唯一改动行：
import { wireMain } from './wiring'      // 旧: from './main-wiring'
```

`wireMain` API 不变，main.ts 调用形态不变。**对 main.ts 是零侵入 + 1 行 import 改动**。

### 4.4 删 main-wiring.ts

`src/daemon/main-wiring.ts` 在拆分后**整文件删除**。`wiring/index.ts` 完全取代。

### 5.2 fake-ilink-server.ts

```ts
export interface FakeIlinkHandle {
  baseUrl: string
  port: number
  /** queue 一个 raw update 让下次 polling 拉到 */
  enqueueInbound(update: RawUpdate): void
  /** 等到 outbox 满足 predicate（轮询，5s 超时）*/
  waitForOutbound(predicate: (msgs: OutboundMsg[]) => boolean, timeoutMs?: number): Promise<OutboundMsg[]>
  /** 当前 outbox 快照（不阻塞）*/
  outbox(): readonly OutboundMsg[]
  /** stop server, free port */
  stop(): Promise<void>
}

export function startFakeIlink(): Promise<FakeIlinkHandle>
```

实现 4 个 endpoint：
- `POST /ilink/bot/getupdates` → 返回 queued RawUpdate[]，然后清空 queue
- `POST /ilink/bot/sendmessage` → 捕获到 outbox
- `POST /ilink/bot/sendfile` → 捕获到 outbox
- `POST /ilink/bot/typing` → no-op，返回 ok

### 5.3 fake-sdk.ts

```ts
export interface FakeSdkScript {

export function installFakeClaude(script: FakeSdkScript): { uninstall(): void }
```

### 5.4 harness.ts

```ts
export interface TestDaemonOpts {
  claudeScript?: FakeSdkScript
  codexScript?: FakeSdkScript
  dangerously?: boolean
  accounts?: IlinkAccount[]
  access?: AccessConfig
  companion?: Partial<CompanionConfig>
}

export interface DaemonHandle {
  ilink: FakeIlinkHandle
  stateDir: string
  sendText(chatId: string, text: string): void
  waitForReplyTo(chatId: string, timeoutMs?: number): Promise<OutboundMsg[]>
  stop(): Promise<void>
}

export function startTestDaemon(opts?: TestDaemonOpts): Promise<DaemonHandle>
```

每个 e2e 套路：
```ts
it('happy path: text in → claude reply out', async () => {
  const daemon = await startTestDaemon({
    claudeScript: { onDispatch: async () => ({
      toolCalls: [{ name: 'reply', input: { text: 'hello back' } }],
      finalText: '',
    }) },
  })
  try {
    daemon.sendText('chat1', 'hi claude')
    const replies = await daemon.waitForReplyTo('chat1')
    expect(replies[0]?.text).toBe('hello back')
  } finally { await daemon.stop() }
})
```

```ts
import { test as base } from '@playwright/test'
import { spawn, type ChildProcess } from 'node:child_process'

interface ShimFixtures {
  shimUrl: string
  shim: { invoke(cmd: string, args?: unknown): Promise<unknown> }
}

export const test = base.extend<ShimFixtures>({
  shimUrl: async ({}, use) => {
    const proc = spawn('bun', ['test-shim.ts'], {
      cwd: 'apps/desktop',
      env: { ...process.env, WECHAT_CC_DRY_RUN: '1', PORT: '5173' },
      stdio: 'pipe',
    })
    await waitForUrl('http://127.0.0.1:5173', 10_000)
    await use('http://127.0.0.1:5173')
    proc.kill('SIGTERM')
  },
  shim: async ({ shimUrl }, use) => {
    await use({
      invoke: async (cmd, args = {}) => {
        const r = await fetch(`${shimUrl}/__invoke`, {
          method: 'POST',
          body: JSON.stringify({ cmd, args }),
        })
        return r.json()
      },
    })
  },
})
export { expect } from '@playwright/test'
```

每个 spec `import { test, expect } from './fixtures'`，自动拿到启好的 shim + 直调 `/__invoke` 的能力。

## 9. 长期扩展（v0.5 不做）

1. **per-chat 动态 permissionMode 查表** —— 当前 bootstrap 算一次静态值，理想是按 chat 动态 lookup。需要 thread `permissionMode` 通过 coordinator dispatch 到每次 makeCanUseTool 调用。v0.6 单独立项。

4. **Tauri 原生 shell e2e** —— 需要 `tauri-driver` + WebDriverIO 桥，跨平台 CI 复杂。shim Playwright 是合理替代。
