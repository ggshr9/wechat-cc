### Dedup semantic（#16）

```ts
const aw = awaiting.get(msg.chatId)
if (aw) {
  if (now - aw.since < 1500 && msg.text === aw.triggerText) {
    deps.log('ONBOARDING', `dedup chat=${chatId} (same trigger text within ${now - aw.since}ms)`)
    return true   // consumed but no name set
  }
  // else: treat as nickname (existing path)
}
```

**Edge case**: user 真发了一字昵称 "x" 在 1.5s 内 → 因为 text ≠ triggerText → 不 dedup → 正常落库。

### `botNameForMode` 实现

```ts
export function botNameForMode(
  mode: ConversationMode,
  registry: ProviderRegistry,
  defaultProviderId: string,
): string {
  const nameOf = (id: string) => id === 'claude' ? 'cc' : id  // codex → 'codex'
  switch (mode.kind) {
    case 'solo':         return nameOf(mode.provider)
    case 'primary_tool': return nameOf(mode.primary)
    case 'parallel':
    case 'chatroom':     return 'cc + codex'
  }
}
```

策略：`claude` → `cc` 别名只在 user-facing string 里替；内部 provider id 仍是 `claude`。

### Onboarding 流程

```
T=0    user "你好" → bot "你好呀！我是 cc，先问一下..." + awaiting.set(chat, {since, triggerText:"你好", fromMessage})
T=N    user "Nate" → name_store.set(chat, "Nate") + reply "好的 Nate, 刚才你说「你好」，回答下："
                  → dispatchInbound(awaiting.fromMessage) → provider 回复正常走 pipeline → 第二条 reply 异步到达
                  → awaiting.delete(chat)
```

**潜在坑**：provider 慢（10s+）→ user 怀疑卡了。**缓解**: ack reply 立刻发，dispatch 异步走，第二条 reply 自然到达。

**Slash command 边界**：第一条消息如果是 `/cc` `/codex` `/help` 等斜杠命令，mode-commands / admin-commands middleware 会在 onboarding gate 之前 short-circuit（既有 `inbound/build.ts` middleware 顺序），所以 onboarding 根本不触发——预期行为，无需特别处理。

### `/whoami` 输出

```
🪪 你: Nate
   WeChat: 张三 (wxid_abc1...)
🤖 bot account: 8ca10d158998... (im-bot)
   当前回应: cc (solo · Claude)
💬 chat: c4b7...
```

读取：`name_store.get(chatId)`、`InboundMsg.userName/userId/accountId`、`coordinator.getMode(chatId)`、`botNameForMode(...)`。
chatId 沉到末尾，调试用；身份信息（你 / bot）置顶。

### `/name <nick>`

- 解析 `tail` 整段当昵称（允许空格）；空 tail → 报 usage `❓ 用法：/name <昵称>`
- 写 `name_store.set(chatId, nickname)`
- reply `✅ 好的，以后叫你 {nickname}。`
- 不做长度/字符严格校验（trim + non-empty 即可）

### Testing

- `bot-name.test.ts`: 6 个 mode 的 mapping 表（含 unknown provider id fallback "wechat-cc"）
- `onboarding.test.ts`:
  - happy path 用新文案
  - 新增 "echo dispatch": mock `dispatchInbound`，2-step onboarding 完成后断言被调用且参数是 trigger msg
  - awaiting 状态在 dispatch 后清空
- `mode-commands.test.ts`:
  - `/name Nate` → name_store.get(chat) === "Nate" + reply 含 "Nate"
  - `/name`（空）→ reply 含 "用法"
  - `/whoami` 已设昵称 → reply 含 nickname + chatId 前缀 + "cc"
  - `/whoami` 未设昵称 → reply "你还没告诉我..."

### Error handling

- `botNameForMode` exhaustive switch，TS 编译期保证；runtime fallback `'wechat-cc'`
- echo dispatch provider 抛错 → onboarding 不重做（昵称已落库），错误走正常 pipeline error path

### 新 PowerShell stop snippet

```powershell
$task = 'wechat-cc'
Stop-ScheduledTask -TaskName $task -ErrorAction SilentlyContinue

$procs = Get-CimInstance Win32_Process -Filter "Name = 'bun.exe' OR Name = 'node.exe'" |
         Where-Object { $_.CommandLine -match 'wechat-cc' }
foreach ($p in $procs) {
  try {
    $h = Get-Process -Id $p.ProcessId -ErrorAction Stop
    if (-not $h.WaitForExit(5000)) {
      Stop-Process -Id $p.ProcessId -Force
      $h.WaitForExit(3000) | Out-Null
    }
  } catch { }
}

$deadline = (Get-Date).AddSeconds(5)
while ((Get-Date) -lt $deadline -and (Get-ScheduledTask -TaskName $task).State -eq 'Running') {
  Start-Sleep -Milliseconds 200
}
```

### task XML 必备字段

```xml
<Settings>
  <AllowHardTerminate>true</AllowHardTerminate>
  <ExecutionTimeLimit>PT0S</ExecutionTimeLimit>
  ...
</Settings>
```

### Dashboard `restartDaemon` 流程

```js
async function restartDaemon() {
  const before = await invoke('wechat_daemon_pid')
  await invoke('wechat_cli_json', { args: ['service', 'restart', '--json'] })
  await sleep(2000)
  const after = await invoke('wechat_daemon_pid')
  if (before && after && before === after) {
    showError('未能重启 daemon — pid 没换。可能是权限问题（dashboard 不是管理员启动）。试试：彻底关闭 → 右键以管理员身份打开。')
    return
  }
  if (!after) {
    showError('daemon 没起来 — 看 install-progress 或 logs 排查。')
    return
  }
  showOk(`✅ daemon 已重启 (pid ${before ?? '?'} → ${after})`)
}
```

### Error handling

- `Stop-Process -Force` 拿到 Access Denied (0x5) → PowerShell 退出码非 0 → CLI 透传 stderr → dashboard 显示"权限不足"
- daemon 没起来 (after pid = null) → 不要默默成功
- 非 Windows 平台 `wechat_daemon_pid` 返回 `null` → dashboard 跳过 pid 检查（其他平台用 systemd/launchd 反馈即可）
