# v1.2.0 (2026-04-24)

可靠性收口版。3.5 天完成，比原预算 2-3 周短得多——less-is-more 哲学落地检验通过。

主题：**消灭 silent failure + 给 Claude 装上翅膀（memory-first Companion v2）**。

## Highlights

- **Companion v2 (memory-first)** — 删掉 v1.1 的 28 字段 RelationshipRecord 设计 + persona/trigger CRUD，改成「沙盒 `memory/` 目录 + 3 个 fs 工具 + scheduler tick」。Claude 自治文件组织、自治判断是否 push。无硬编码规则。Karpathy LLM wiki 哲学 + Tesla FSD v11→v12 思路（"删规则，信任模型"）。
- **会话 resume 跨 daemon 重启** — 持久化 `alias → session_id` map (`sessions.json`)；spawn 时优先传 `resume: <session_id>` 给 SDK；jsonl 不在或 7 天过期自动 fall back 到 fresh spawn。重启后首响从 ~10s 冷启动降到 <3s。
- **死 bot 诊断工具链** — `errcode=-14` session timeout 检测 → `SessionStateStore` 持久化 flag → `/health` admin 命令展示活跃 + 过期 bot → `清理 <bot-id>` / `清理所有过期` 清理命令。用户明确选 pull-based（查询才告警），不主动推。
- **Silent-failure 两连修**：
  - `reply_voice` WAV 原走 `voice_item` shape（微信客户端静默丢弃），现回落到文件附件——至少收得到能播放的 .wav。`[VOICE]` 日志同步上线。
  - `[SESSION_EXPIRED]` / `[TYPING]` 之前只走 stderr，log-viewer 看不见；搬到 `channel.log`。

## Breaking changes

- **Companion v1.x 状态被 v2 忽略**：`companion/config.json` 里的 `personas[]` / `triggers[]` / `per_project_persona` 字段在 v2 下不读不写。用户若之前开了 Companion 并存过 persona/trigger，重启 daemon 后这些字段静默失效，其余配置（enabled / timezone / default_chat_id / snooze_until）保留。
- **5 个工具删除**：`persona_switch`、`trigger_add`、`trigger_remove`、`trigger_pause` 不再可用；`companion_status` 返回字段简化（只剩 enabled / timezone / default_chat_id / snooze_until）。
- **无数据迁移**。memory/ 目录自动创建在首次 `memory_write` 时。

## Internals

- **`ilink-glue.ts` 拆包** — 558 行单 closure → 263 行组合根 + `src/daemon/ilink/{context,voice,companion,transport}.ts` 四个 feature 模块。公共 API 零变更（`makeIlinkAdapter` / `loadAllAccounts` / `parsePermissionReply` / `Account`）。
- **`src/daemon/memory/fs-api.ts`** — 沙盒 FS：.md only / 100KB 单文件 / 路径校验（拒绝 `..` / 绝对路径 / null-byte）/ atomic rename / 0o600 mode。21 测试通过。
- **`src/core/session-store.ts`** — persistent alias map，500ms debounced write，atomic rename，7 天 TTL。
- **`session-manager.ts`** — 新增可选 `sessionStore` + `canResume` + `resumeTTLMs` ctor 参数；spawn 时读 store + 校验 jsonl 存在，过期或缺失则 delete 记录；result message 到达时 `store.set(alias, session_id)`。
- **Scheduler** — 从 croner cron 简化到 setInterval + jitter。tests rewritten with fake-timer compatible shape (100ms floor).
