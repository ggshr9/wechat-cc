# wechat-cc desktop v0.5.2

**Date**: 2026-05-05
**Tag**: `desktop-v0.5.2`
**Scope**: Sidecar binary rebuild against CLI v0.5.2 — fixes Windows ScheduledTask install (`-AllowHardTerminate` cmdlet-parameter bug) that blocked every Win11 install since `desktop-v0.5.1`. No desktop UI changes.

> If your `desktop-v0.5.1` install hit `New-ScheduledTaskSettingsSet : A parameter cannot be found that matches parameter name 'AllowHardTerminate'` at wizard step (1/2) — this is the bundle that fixes it. Re-download for your platform; no state migration needed.

## What changed

The compiled `wechat-cc-cli.exe / cli` sidecar shipped inside this bundle is built from CLI v0.5.2, which contains:

1. **Win install hotfix** — `$settings.AllowHardTerminate = $true` is now set as a property on the `MSFT_TaskSettings3` CIM object instead of a (non-existent) `-AllowHardTerminate` cmdlet parameter on `New-ScheduledTaskSettingsSet`.
2. **PowerShell binding validator in CI** — every install/start/stop/uninstall script generated by `service-manager.ts` is parsed and parameter-bound on the windows-latest CI runner before any PR can merge. The pattern that let this bug — and >10 similar Win regressions over the past year — ship green is now closed.

Linux + macOS sidecars are byte-equivalent to `desktop-v0.5.1` (the source delta only touches the win32 branch of `service-manager.ts`); rebuilding them keeps all three platforms version-aligned.
