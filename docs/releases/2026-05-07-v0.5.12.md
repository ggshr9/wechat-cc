# v0.5.12 — codex sessions render in the dashboard 会话 tab

**Date**: 2026-05-07
**Tag**: `v0.5.12` + `desktop-v0.5.12`
**Scope**: Lift the v0.5.11 limitation by actually reading codex rollouts. **No breaking changes.**
**Predecessor**: [v0.5.11 — sessions detail no longer breaks for codex sessions](2026-05-07-v0.5.11.md)

---

## What v0.5.11 left on the table

v0.5.11 fixed the cryptic ENOENT and explained the limitation:

> session was last touched by codex; jsonl viewer only supports claude sessions today

…which was honest but unsatisfying. Users testing `/codex` or `/chat` saw the `compass` project's session as a dead-end in the dashboard.

## What v0.5.12 does

`sessions read-jsonl` now reads codex rollouts and emits claude-shape turns so the existing dashboard renderer works unchanged.

| | |
|---|---|
| Path resolver | New `findCodexRollout(codexRoot, threadId)` walks `~/.codex/sessions/<YYYY>/<MM>/<DD>/` for files matching `*-<threadId>.jsonl`. Bounded to depth-3 (codex's known sharding scheme) so it can't accidentally scan unbounded user dirs |
| Event-shape converter | New `readCodexJsonlAsClaudeTurns(path)` parses each line, picks `response_item` entries with `payload.type === 'message'`, maps `role: user/assistant` → claude-shape `type: user/assistant`, maps `content[].input_text/output_text` → `content[].text`. Drops `session_meta`, `reasoning`, and `event_msg` (latter duplicates `response_item.message.assistant`) |
| CLI integration | `cli.ts sessions read-jsonl` now branches on `rec.provider`. Codex sessions → codex helpers. Claude sessions → existing path. Frontend response shape unchanged (`{ ok, alias, session_id, turns }` plus an extra `provider` field on codex) |

The dashboard renderer (`turnHtml*` family) consumes turns by `turn.type === 'user' | 'assistant'` and walks `turn.message.content[*].text` — both paths now feed it the same shape.

## What you'll see

Verified live on the maintainer's daemon: clicking into the `compass` project (whose most-recent session is codex's `019e005a-...`) now renders **11 turns** in the iPhone-style chat. Previously: `读取失败：session was last touched by codex...`. Now: actual conversation bubbles.

Caveats:
- For chatroom sessions, codex's "user" inputs are moderator-generated prompts (full system blob + chatroom envelope), not the raw wechat msg. Showing them as user-bubbles is technically accurate (it's what codex saw) but may look strange. Filtering moderator prompts is a future polish.
- Reasoning items are dropped (encrypted summaries aren't human-readable anyway).
