## ✨ 新功能

- **WeChat 端 `/restart`**：微信里发 `/restart` 即可重启 wechat-cc，自动通过 Claude Code 的开发通道确认对话框，重启完成后通过微信发送「已重连，用时 N 秒」通知。不用再跑回电脑按回车。支持 `--fresh`（开新 session）和 `--dangerously`（跳过权限）后缀；iOS/WeChat 自动把 `--` 改成 em-dash 的 case 已被反向修正。
- **`share_page` / `resurface_page` 长文档分享**：Claude 可以把 plan / spec / 审阅文档发布成手机浏览器直接点开的渲染页面（走 cloudflared quick tunnel），带代码块、表格、列表样式。`resurface_page` 可以把磁盘上的旧文档重新挂到当前 tunnel 上生成新 URL。发布内容保留 7 天。首次调用自动下载 cloudflared 静态二进制到 `~/.claude/channels/wechat/bin/`，无需手动 `brew install cloudflared`。
- **share_page 一键 Approve**：页面底部有一个 Approve 按钮，可以把 URL 转发给非 Claude 生态的同事/上司审阅，他们点击后 Claude 会以 `[share_page 审阅]` inbound notification 形式收到「已通过」反馈（chat_id 以 `share_page:<slug>` 标记，便于区分于普通微信消息）。
- **`wechat-cc update` 一键升级**：`git pull --ff-only` + 条件 `bun install`（只在 `bun.lock` 变化时重装依赖）。WeChat 端 `/status` 同步升级：显示当前版本 SHA + 最新 commit subject + 是否落后于 `origin/master`，远端检查结果缓存 5 分钟。
- **紧凑权限转发 + 裸 y/n 批复**：Claude Code 的工具权限对话框（🔐）改为紧凑格式发到微信，可直接回 `y` / `n` 批复当前唯一待定请求，或回 `/perm <code>` 查看完整详情。
- **`reply` 工具支持 `files[]`**：一次调用可以同时发送文本 + 多个附件（图片/视频/文件），无需多次往返。

## 🐛 修复

- 图片发送后显示为灰块：修正 `aes_key` 的 base64 编码格式（应为 `base64(hex-string)` 而非 `base64(raw-16-bytes)`）。
- `pollLoop` 的消息丢失竞态：`saveSyncBuf` 挪到 `handleInbound` 之后，进程崩溃不会再让未处理消息被 ilink 侧标记为已送达。
- 重启后 `broadcast` 广播空手而归：改用磁盘持久化的 `contextTokens` 作真相源，不再依赖运行时只在内存里的 `userAccountMap`。
- 重启后权限对话框静默消失导致 Claude 卡死：`userAccountMap` 在 `mcp.connect` 之前初始化 + permission handler 加 `_startupAccounts` 兜底路由。
- `access.json` 损坏时的静默回退：改为 FATAL + `process.exit(1)`，不再无声地把所有合法用户锁在白名单外。
- 未授权消息被丢弃时增加 `[GATED]` 日志行，便于排查「为什么某人的消息没进来」。
- 语音消息内容缺失时 Claude 不再靠上下文猜测：新指令要求在看到「语音 · 未转文字 / 空 / 下载失败」等标记时，立即回复用户「我没拿到语音内容，请打字或重发」，而不是当作空气。
- 权限对话框 `input_preview` 被 Claude Code 预截断导致字段解析失败：改用 regex 从截断原文里提取。
- `/restart` SIGTERM 前 flush `context_tokens.json`，避免在 3 秒 debounce 窗口里更新过的 token 被带走。

## ⚡ 质量改进

- 文件保留 TTL：`docs/`（share_page 发布）7 天自动清理，`inbox/`（收到的媒体）30 天自动清理，`channel.log` 超 10MB 自动 rotate 成 `channel.log.1`。
- 小文本类文件入站自带前 5 行预览：小于 10KB 的 `.csv` / `.md` / `.txt` / `.json` / `.yml` 及常见代码文件，入站 notification 里自动附带文件大小、总行数、前 5 行内容，免去 Claude 再次 Read 的多余往返。二进制伪装成文本（控制字符占比 >10%）的会跳过预览。
- CDN 上传对临时 abort / 5xx 自动重试 3 次，间隔 1 秒。
- `/restart` 相关：重启路径用 `expect(1)` 包装 `claude`，通过三次定时盲送回车自动通过开发通道对话框。`expect` 作为可选依赖，没装也能跑，只是 `/restart` 需要有人在终端手动按回车。

## ⚠️ 注意事项

- **MCP tool 重命名**：`share_doc` → `share_page`。v0.1.0 里还没有 `share_doc`，所以直接从 v0.1.0 升级到 v0.2.0 对你来说是**新增**而非变更；但如果你手头有基于 0.1.0 → 0.2.0 之间任何中间 commit 的集成代码，并引用了 `share_doc` 这个名字，请改为 `share_page`。同时对应的内部函数名也从 `shareDoc` 改成了 `sharePage`，类型 `ShareDocResult` → `SharePageResult`。
- **升级路径**：因为 `wechat-cc update` 本身是 v0.2.0 引入的，第一次从 v0.1.0 升级仍然需要手动：

  ```bash
  cd ~/.claude/plugins/local/wechat
  git pull
  bun install          # 如果 bun.lock 有变化
  ```

  重启：在 WeChat 发 `/restart`（需要装 `expect` 以跳过开发通道对话框，否则要在终端手动按一次回车）。

- **可选依赖**：`expect`（用于 `/restart` 自动通过对话框）、`cloudflared`（用于 `share_page`，首次调用时自动下载）。两者都不装也能工作，只是相关功能的 UX 会退化。
