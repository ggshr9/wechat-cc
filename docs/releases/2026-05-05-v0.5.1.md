## 行为等价 + Migration

老用户升级路径：
1. 装上新 daemon 启动一次 → 自动跑 DB v9 ALTER（加 3 列）+ 一次性 backfill `user_names.json` → SQLite → 文件改名 `.migrated`
2. 后续 inbound 通过 `mw-identity` 自动维护身份列
3. **不需要重新扫码、不需要清状态、不需要任何手动操作**

如果你对老 `user_names.json` 还有依赖（不该有），它就在 `<stateDir>/user_names.json.migrated` 留着，可以读。

## 桌面 UX 视角的具体变化

### 1. 会话切换 / 控制台
- conversations 表头从 `用户 / Chat ID / 模式` 变 `用户 / 账号 / 模式`
- 用户列：`{userName} ({wxid 截断})` 而不是裸 chatId
- 账号列：truncated accountId（hover 看完整）
- chatId 移到 row 的 `title` tooltip（开发调试用）

### 2. 账号 expired 体验
- 新增桌面 OS 通知：`wechat-cc: 账号 X 已失效`，body：`绑定被另一处替换。点击打开 dashboard 重新扫码。`（依赖新加的 `tauri-plugin-notification`）
- accounts 表 expired 行 + "重新扫码" 按钮 → 点击跳到 wizard bind 步骤
- in-chat 通知（每个曾经发过消息的 chat 都收到一条）：
  ```
  ⚠️ 这个 WeChat 账号的绑定已被替换 —— 你或其他人在别处重新扫码绑了同一个账号到 wechat-cc / OpenClaw。
  本机的旧 session 已失效，不会再收到新消息。
  要继续在这台机用，到桌面 dashboard 重新扫一次码。
  ```

### 3. Windows 重启 daemon
- 现在点 dashboard "重启 daemon" 真的杀进程并重启
- 没用管理员启动 dashboard 时点重启 → 立刻提示"权限问题，试试管理员打开"，不再"假装成功"
- daemon 没起来 → "daemon 没起来 — 看 install-progress 或 logs 排查"，不再静默
- ScheduledTask XML 现在含 `<AllowHardTerminate>true</AllowHardTerminate>` —— 关键，没有这个 `Stop-ScheduledTask` 只是个软提示

### 4. Onboarding 体验
首次扫码绑定后第一次发消息：
```
user: 你好
bot:  你好呀！我是 cc，先问一下我应该怎么称呼你？比如「Nate」「丸子」（中文 / 英文都行）。
user: Nate
bot:  好的 Nate, 刚才你说「你好」, 回答下：
bot:  你好啊! 有什么我可以帮的？  ← provider 真正回答（异步）
```

新加 `/name 张三` 改昵称、`/whoami` 看身份 dump（包含 nickname / WeChat 名 / accountId / 当前 mode 对应的 bot 名 / chatId）。
