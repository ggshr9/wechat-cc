# v0.6.0 — Multi-provider tiers, dashboard reconnect/diagnose, companion proactivity

**Date**: 2026-05-29
**Tag**: `v0.6.0` + `desktop-v0.6.0`
**Scope**: The first **minor** release since the v0.x reset — 234 commits since v0.5.18. It lands the multi-provider permission model (RFC 05: Cursor as a third agent, user tiers, per-chat session isolation), a dashboard **reconnect/diagnose** flow + live provider switching, a guided 4-step setup wizard, **Companion proactivity v1** (the companion now follows through on self-authored, time-anchored intentions instead of cold-polling), and a wide band of reliability/security hardening. Plus dev-infra: the companion eval harness and desktop Playwright e2e (now in CI).
**Predecessor**: [v0.5.18 — Post-review hotfix + single-page setup](2026-05-18-v0.5.18.md)

---

## TL;DR

1. **Multi-provider + tiers (RFC 05).** Cursor joins Claude + Codex as a first-class agent. A three-tier permission model (admin / trusted / guest, from `access.json`) gates tools per chat, on top of the daemon-wide strict/`--dangerously` mode. Per-chat session isolation means `/cc` and `/codex` no longer share or clobber each other's session.
2. **Dashboard reconnect/diagnose + live provider switch.** When the daemon looks unhealthy, the dashboard surfaces a diagnose card that maps the failure to a concrete next action and can drive a restart; a provider chip lets you switch claude/codex/cursor with an auto-restart.
3. **Guided 4-step setup wizard** (env check → agent pick → QR → service), with Cursor as a selectable third agent.
4. **Companion proactivity v1.** The companion records dated follow-ups in `agenda.md` during conversation and the push tick *fires the due ones* (default = act) instead of waking cold every 20 min to ask "should I say something?". Cheaper too — no LLM call on a tick with nothing due.
5. **Reliability & security hardening** across the dispatch path, persistence, memory sandbox, inbound access gate, and the desktop attack surface.

## Multi-provider & permission tiers (RFC 05)

- **Cursor as a third agent** — selectable in the wizard and dashboard; `/cursor` solo command; `/chat <p…>` and `/parallel <p…>` accept explicit participant lists.
- **User tiers** — `admin` / `trusted` / `guest`, resolved from `access.json`, orthogonal to the daemon-wide permission mode. `TIER_PROFILES` + `classifyToolUse` + a tier-aware `canUseTool` relay decide allow / relay-to-admin / deny per tool, per chat.
- **Per-chat session isolation** — every session query is keyed by `chatId`; the session manager acquires per-chat, so concurrent chats (and `/cc` vs `/codex`) don't share or evict each other.
- **Provider-agnostic cheap-eval** — Codex-only users are fully supported; chatroom moderation / companion introspect pick the cheapest registered provider and are model-rot-proof.
- **Delegate tier policy** — `delegate_<peer>` runs the peer at a bounded tier (Codex peer read-only / workspace-write as appropriate), one-shot, depth-capped.

### ⚠️ Behavior change — admin destructive ops now prompt in strict mode

Folds in [2026-05-26 tier-policy change](2026-05-26-tier-policy-change.md): under the default **strict** mode, an admin's `shell_destructive` (`rm -rf`, `git reset --hard`, …) and `memory_delete` now **relay a prompt to the admin's own chat** instead of auto-running. Everything else for admin/trusted still auto-allows. `--dangerously` is unchanged (full SDK-level bypass for all tiers). This is the deliberate fix for the brief v0.5.x window where "admin == dangerously".

## Dashboard

- **Reconnect / diagnose card** — a pure `diagnose()` maps daemon health (codes 1–8) to a specific cause + primary/secondary action; `runRestartSequence` can drive the fix; `RECONNECT_DIAGNOSE` telemetry on every reconnect; `wechat_health_ping` Tauri command + `pingHealth` wrapper.
- **Provider chip dropdown** — switch claude / codex / cursor from the dashboard with an auto-restart.
- Admin row now reads `access.json.admins[]` (not `accounts[0]`); honest activity-time copy (no fabricated timestamps).

## Setup wizard

- Adopted the **4-step guided wizard** (doctor → provider → wechat → service) with a refreshed design and logo.
- Cursor card as a third agent option; inline hint + flash CTA when the first agent is detected; per-provider missing-leg copy.

## Companion proactivity v1

- The companion authors dated follow-ups into `memory/<chat>/agenda.md` (`- [ ] due:YYYY-MM-DD …`) during conversation; the push tick parses the agenda, **fires the oldest due item** (default = act, with a narrow skip-if-stale), and marks it done — firing each at most once.
- A tick with nothing due returns **silently without an LLM call** (cheaper than the old every-tick evaluation).
- **Current-time authority** — inbound envelopes carry an ISO-8601 `ts` and the prompt declares it the authoritative "now", so date reasoning ("下周三") is anchored to the conversation, not a stale baseline.
- Known limitation (below): the companion is restraint-biased about proactive outreach by disposition; it won't always resurface an overdue thread.

## Bot identity & help

- Real `/help` (= `/帮助`) command — user-friendly command list, with an extra admin section.
- `/botname` admin command (renamed from the old `/name`, which is now the user self-rename) + two-step onboarding that also asks the bot's display name.

## Reliability & security hardening

- Closed three tier-policy regressions in the v0.6 dispatch path and four silent data-loss bugs (conversation / agent-config / A2A wiring).
- `memory/` FS now rejects symlink-escape on read/write/delete/list; inbound enforces the `access.json` allowlist gate and fires correctly even with an unknown IP; chatroom isolates per-dispatch history + recovers orphan-user turns.
- Internal API handles CORS preflight without auth; the desktop dev-server/shim closed an `/attachment` LFI and hardened `</body>` injection (Windows-safe).
- Provider robustness: codex binary lookup prefers the daemon repo's `node_modules`; cursor refuses registration when its model is missing; SDK lifecycle errors from a dead claude subprocess are swallowed; `windowsHide` on async spawns.

## Dev / test infra

- **Companion eval harness** — all 8 failure modes have trajectories (multi-contact supported for cross-chat isolation); the claude-sdk judge resolves the binary so dimension scores run.
- **Desktop Playwright e2e now runs in CI** (`desktop-e2e` job) — 96 specs covering wizard, dashboard, CSP, attachment guard; CSP injection defaults **on** so the security control is never silently skipped.

## Known limitations

- **Companion proactive push is restraint-biased.** Even with a due, persisted follow-up, the companion often chooses silence — this is persona-level disposition, not a bug (over-pushing a companion is worse than under-pushing). The fire mechanism is unit-proven and production-correct.
- **Time-anchored companion e2e isn't deterministically assertable** — the eval replays virtual time but the agent's SDK uses the real clock, so `long_silence` / `tech_stress` proactive probes are `decision: n/a`. Closing this needs virtual-time propagation to the agent (deferred).
- **No Rust unit tests** for the Tauri command layer yet (deferred — needs a Rust toolchain / CI job).
- Carries forward v0.5.x limitations: first-contact required before the bot can message; ilink is 1:1 only; macOS Intel desktop bundle not provided; desktop bundle unsigned (first-launch Gatekeeper/SmartScreen bypass).

## Upgrade notes

- No config migration required. `sessions.json` / `wechat-cc.db` carry over.
- If you share your daemon via `access.json.allowFrom[]`, note that non-admin chats are now **guest** tier by default (reply / share_page / memory_read / observations_read only) — promote trusted users in `access.json`.
- Admins on strict mode will now see prompts for destructive Bash / `memory_delete` (see the behavior-change note above); run with `--dangerously` to opt the whole daemon out.
