## One-brain-many-hands (乙)

A stable machine that holds the WeChat bot (the **brain**) can delegate tasks to other machines running wechat-cc (the **hands**). A hand exposes an authenticated `/a2a/exec` endpoint that runs its local agent (Read/Bash) and returns the result; the brain dispatches via the `让<name>执行` trigger and relays the reply to chat. This lets you drive your home machine — or query a project that only lives there — from the office.

**Smooth pairing.** The hand mints a one-time code (`hand invite`): a short-lived (10-min), single-use secret + its tailnet A2A URL. The brain joins (`hand join <code> --id home --name 家里`), which auto-registers both sides via a `/a2a/pair` callback — no manual token copying. `hand list` shows paired hands/brains; `hand ping` checks reachability.

**Security.** The A2A server is off by default and binds `127.0.0.1` unless overridden; run it only on a private Tailscale IP. `/a2a/exec` requires a per-agent Bearer key (constant-time compared); the pairing secret is one-time, expiring, and consumed only on success. The exec key is effectively a remote-shell credential — keep it on your tailnet.

## Work + life memory

`整理记忆` reads `~/.claude/projects/*/memory/` (work) plus the companion's observations / milestones / notes (life) and synthesizes a single `_overview.md` the bot folds into its context. `看记忆` / `你对我的理解` reads it back (stamp stripped, timestamp surfaced). The terminal mirrors this: `wechat-cc memory synthesize` regenerates it, `memory status` shows freshness + how much source memory is available, `memory projects` lists the work-side source.
